# Шаблоны отчётов PSS-C PDM

## Как добавить новый отчёт

### 1. Создайте HTML-файл шаблона

Создайте файл `*.html` в этой директории (`PSS-C/reports/`). Имя файла должно отражать назначение отчёта, например `my_report.html`.

### 2. Добавьте метаданные в начало файла

В начало файла обязательно добавьте HTML-комментарии с метаданными:

```html
<!-- REPORT: Название отчёта -->
<!-- DESCRIPTION: Описание назначения отчёта -->
<!-- PARAMS: param1 (тип) - описание параметра -->
```

- **REPORT** — краткое название отчёта (отображается в меню).
- **DESCRIPTION** — описание того, что показывает отчёт.
- **PARAMS** — входные параметры. Каждый параметр на отдельной строке `<!-- PARAMS: ... -->`.

### 3. Используйте Jinja2-синтаксис

Шаблоны рендерятся движком [Jinja2](https://jinja.palletsprojects.com/). Основные конструкции:

```html
{{ variable }}                    — вывод переменной
{{ variable or '—' }}             — значение по умолчанию
{% for item in list %}...{% endfor %}  — цикл
{% if condition %}...{% endif %}       — условие
{{ loop.index }}                  — номер итерации (начиная с 1)
```

### 4. Контекст рендеринга

Переменные контекста передаются при вызове отчёта. Документируйте ожидаемую структуру данных в комментариях шаблона. Пример:

```html
<!--
  Context variables:
  - items: [{name, code, quantity}]
  - title: str
-->
```

### 5. Стилизация для печати

Для корректной печати добавьте CSS-блок `@media print`:

```css
@media print {
    body { margin: 0; font-size: 10pt; }
    .no-print { display: none; }
    table { page-break-inside: auto; }
    tr { page-break-inside: avoid; }
}
```

### 6. Существующие шаблоны

| Файл                  | Назначение                                   |
|-----------------------|----------------------------------------------|
| `bom_report.html`     | Структура изделия (BOM) — иерархический состав |
| `process_report.html` | Техпроцесс — операции, документы, материалы    |

### 7. Рекомендации

- Используйте единый стиль оформления таблиц (см. существующие шаблоны).
- Все текстовые строки — на русском языке.
- Предусматривайте пустые состояния (`{% if not items %}`).
- Указывайте `lang="ru"` в теге `<html>`.
- Кодировка файлов — UTF-8.
