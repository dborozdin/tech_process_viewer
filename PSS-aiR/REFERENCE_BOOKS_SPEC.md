# Спецификация раздела "Справочники" в PSS-aiR

## Назначение

Раздел "Справочники" предоставляет доступ ко всем справочным таблицам PSS (Product Data Management System) через единый интерфейс. Основные цели:

1. **Централизованный доступ** ко всем системным справочникам PSS
2. **Иерархическое отображение** классификаторов (системы и уровни) с поддержкой древовидной структуры
3. **Единообразный интерфейс** для работы со справочными данными
4. **Быстрый поиск** и навигация по справочной информации

## Статус реализации (2026-04-20, переработано)

Раздел "Справочники" переработан с переходом на ленивую загрузку. В тестовой БД
`pss_moma_08_07_2025` 7 систем классификаторов, самая крупная (MOMA) содержит
3863 уровня — поэтому единоразовая загрузка всей иерархии неприемлема.

### ✅ Выполнено и проверено:
1. **Backend `ClassifiersAPI`** (`api/pss_classifiers_api.py`): полностью переписан
   - Убраны несуществующие атрибуты (`is_wire`, `is_flammable`, `flammable_material`
     и пр.) — использованы актуальные поля схемы (`section_type`, `only_construction`,
     `wiring`, `wiring_near_controls`, `combustible_materials`, `name_eng`, `okpd_code`,
     `is_system_level`).
   - Функции `get_root_levels(system_sys_id)` и `get_child_levels(parent_sys_id)`
     для ленивой загрузки.
   - `get_classifier_tree` стал неглубоким (`max_depth=2` по умолчанию).
   - Убраны N+1 запросы на подсчёт потомков у списка систем.
   - `delete_*` используют общий `DatabaseAPI.delete_instance` (HTTP DELETE /rest/{id}).

2. **`ReferenceService`** (`PSS-aiR/services/reference_service.py`): прокси с
   декоратором `@track_performance`, включает методы `get_root_levels` и
   `get_child_levels`.

3. **REST маршруты** (`PSS-aiR/routes/references.py`):
   - `GET /api/references/types` — список типов справочников (включая количество).
   - `GET /api/references/classifiers` — список систем классификаторов.
   - `GET /api/references/classifiers/{sys_id}/tree?max_depth=2` — неглубокое дерево.
   - `GET /api/references/classifiers/{sys_id}/roots` — **корневые уровни системы**.
   - `GET /api/references/levels/{parent_sys_id}/children` — **прямые потомки уровня**.
   - `GET /api/references/classifiers/levels/{sys_id}` — полные детали уровня.
   - `GET /api/references/search?q=...&type=all|systems|levels` — поиск (кириллица
     фильтруется на стороне Python, т.к. APL LIKE с русскими символами не работает).
   - `GET /api/references/list/{type}` — заглушки.
   - CRUD системы: `POST /classifiers`, `PUT /classifiers/{id}`, `DELETE /classifiers/{id}`.
   - CRUD уровней: `POST /classifiers/levels`, `PUT /classifiers/levels/{id}`,
     `DELETE /classifiers/levels/{id}`.

4. **MCP инструменты** (`mcp_server/server.py`): добавлены
   `pdm_get_classifier_roots` и `pdm_get_classifier_children`, обновлено
   описание `pdm_get_classifier_tree`.

5. **Frontend** (`PSS-aiR/static/index.html`): компонент `ReferencesView`
   переписан. Трёхпанельный интерфейс:
   - слева — типы справочников;
   - в центре — дерево систем и уровней с ленивой загрузкой и клик-по-названию
     = «выбрать + авто-раскрыть»;
   - справа — панель деталей (`#referenceDetailsPanel`) с нормальными полями
     уровня (система, родитель, код, секция, технические признаки и т.д.).
   - Исправлены CSS-конфликт с основным деревом навигации
     (правило `.tree-children { display: none }` перекрывалось `.hidden` классом).
   - Исправлена ошибка, из-за которой при клике на кнопку «Справочники» весь `#app`
     (вместе с overlay-панелью) скрывался. Теперь overlay показывается поверх.

6. **Тесты**:
   - `PSS-aiR/test_references_api.py` — 4 PASS.
   - `PSS-aiR/test_references_crud_api.py` — 9 PASS (CRUD систем и уровней).
   - `PSS-aiR/test_references_ui.py` — 11 PASS (Playwright с headless=False).

### Известные ограничения
- PSS APL отдаёт ≤200 экземпляров за один запрос. `get_root_levels` сначала
  загружает все уровни системы (ограничиваясь 200), затем фильтрует — для очень
  крупных систем нужно учитывать, что корневых уровней обычно единицы (в MOMA их 2).
- Кэширование не реализовано (при необходимости добавить TTL-кэш на уровне сервиса).

## Архитектура

2. **Сервис**: `PSS-aiR/services/reference_service.py` — бизнес-логика с декоратором `@track_performance`
3. **Маршруты**: `PSS-aiR/routes/references.py` — REST API endpoints
4. **MCP инструменты**: `mcp_server/server.py` — обновление для работы через Claude Code

1. **Кнопка в заголовке**: "📚 Справочники" в верхней панели рядом с "PSS-aiR"
2. **Основной компонент**: `ReferencesView` — контейнер для всех справочников
3. **Компонент классификаторов**: `ClassifierTree` — древовидное отображение систем и уровней
4. **Табличный компонент**: `ReferenceTable` — отображение простых справочников (заглушки)

### Справочные таблицы PSS

#### Полная реализация
1. **Классификаторы**:
   - `apl_classifier_system` (ID 1400) — системы классификаторов с иерархией через `parent`/`childs`
   - `apl_classifier_level` (ID 1403) — уровни классификаторов с иерархией внутри системы через `parent`/`childs`
   - `apl_classifier_association` (ID 1409) — ассоциации классификаторов
   - `apl_classifier_lang` (ID 1875) — языковые версии классификаторов

#### Заглушки (кликабельные, функционал "в разработке")
2. **Единицы измерения**:
   - `si_unit` — базовые единицы СИ
   - `conversion_based_unit` — производные единицы
   - *Существующий API*: `pss_units_api.py`

3. **Организации**:
   - `organization` — организации-исполнители
   - *Существующий API*: `pss_orgs_api.py`

4. **Характеристики**:
   - `apl_property_definition` — определения характеристик
   - *Существующий API*: `pss_characteristic_api.py`

5. **Другие справочники**:
   - Типы документов
   - Типы ресурсов
   - Статусы объектов
   - Категории

## MCP инструменты

### Обновление MCP сервера
Согласно правилу синхронизации в CLAUDE.md, при изменении классов API в папке `api/` необходимо обновить `mcp_server/server.py`.

**Новые инструменты:**
